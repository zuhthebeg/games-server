## 아키텍처: 하이브리드 (캔버스 테이블 + DOM 버튼)
- **캔버스**: 펠트, 좌석 아치, 카드, 칩, 딜러 사진, 애니(칩 던지기·딜링·홀카드 플립)
- **DOM 오버레이(HUD)**: Hit/Stand/Double/Split·베팅 버튼·이름표 — 탭 타겟/접근성
- 게임 로직 불변(서버권위). 바꾸는 건 렌더 레이어(renderMultiState DOM → 캔버스).

## 룰/흐름 (cocy 확정)
- **베팅 동시** — 순서 없음. 각자 bet→ready, 전원 ready 시 딜. (서버 이미 이 구조)
- **카드 플레이만 턴 순서** — getCurrentTurn(playing)만 순서. AI도 동일.
- AI는 베팅 단계에 runAI가 미베팅 AI를 구동(순서 무관, 즉시 채움).

## 좌석 배치
딜러 상단 중앙(사진+카드). 플레이어 하단 반원 아치 6석. "나"=정면 중앙 큰 카드, 나머지 좌우 감싸기. 인원수별 좌표 동적.

## 단계 (리스크 낮은 순)
- **Phase 0 — 서버 AI** (선행, 비주얼 무관): getAIAction(베팅=랜덤 소액, 플레이=hit<17/stand), createInitialState ai-* 패딩(config.seats), getCurrentTurn 베팅 시 미베팅 AI. → 기존 DOM UI로도 멀티+AI 동작. **← 지금 착수**
- **Phase 1 — 캔버스 골격**: 펠트+좌석 아치, state→좌표 매핑, HUD는 DOM. `?table=canvas` 플래그 토글(DOM 폴백 유지).
- **Phase 2 — 애니**: 카드 딜링 슬라이드(슈→좌석), 칩 던지기, 홀카드 플립, 버스트/BJ 연출.
- **Phase 3 — 딜러 사진 + BGM**: LuminarQMix 딜러 2~3종(웹 SFW) → 캔버스 딜러존. 승/패 표정. ACE-Step 긴장감 BGM 루프.
